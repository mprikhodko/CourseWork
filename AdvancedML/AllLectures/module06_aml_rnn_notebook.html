Introduction

A sequence is an ordered collection of items. Classical approaches, such as N-gram language models, assumes data points to be independently and identically distributed (IID), but in many situations (like with language, speech, and time series data), given one data item, it depends on the items that precede or follow it. For example, speech is a sequence of phonemes. In English language, words in a sentence form sequences as we have seen in previous modules, such as word tagging. As an example, the preposition 'of' is likely followed by the article 'the'.

In deep learning, modeling sequences involves maintaining hidden state information, similar to Hidden Markov Models. As each item in the sequence is encountered, the hidden state is updated. Thus, the hidden state (usually a vector) encapsulates everything seen by the sequence so far. This hidden state vector, also called a sequence representation, can then be used in many sequence modeling tasks such as Word2Vec.

Recurrent Neural Networks

In PyTorch, Recurrent Neural Networks, like Feedforward networks, are a family of models, and they model sequences of tensors. In the following cells, we will use Elman RNNs which one of many available in PyTorch. The representation of a sequence is learned by maintaining a hidden state vector that captures the current state of the sequence. The hidden state vector is computed from both the current input vector and the previous hidden state vector. Below, left figure is the functional and the right figure is the "unrolled" view of the computational dependencies. In the case of an Elman RNN, the hidden vector is what is predicted.

The back-propagation through time (BPTT) is used, where the input vector from the current time step and the hidden state vector from the previous time step are mapped to the hidden state vector of the current time step. A new hidden vector is computed using a hidden-to-hidden weight matrix to map the previous hidden state vector and an input-to-hidden weight matrix to map the input vector. The hidden-to-hidden and input-to-hidden weights are shared across the different time steps.

RNNs use the same parameters to compute outputs at every time step by relying on a hidden state vector to capture the state of the sequence. In this way, the goal of RNNs is to learn sequence invariance by being able to compute any output given the hidden state vector and the input vector. For compare/contrast reference, an RNN shares parameters across time and a CNN (Convolutional Neural Networks) shares parameters across space.

  • RNN $\longrightarrow$ temporal (such as signal, speech, NLP)
  • CNN $\longrightarrow$ spatial (such as images)
  • Then video $\longrightarrow$ CNN + RNN

As words and sentences can be of different lengths, the RNN or any sequence model should be equipped to handle variable length sequences. One possible solution is to restrict sequences to a fixed length artificially. Another technique, called masking takes advantage of knowledge of the lengths of the sequences. In brief, masking allows for the data to signal when certain inputs should not count toward the gradient or the eventual output. PyTorch provides primitives for handling variable length sequences called PackedSequences that create dense tensors from these less dense ones.

Drawbacks of RNNs

Saturation, vanishing/exploding gradients, complex loss surfaces and numerous local minima, not going well with dropout are among the problems with RNNs. LSTMs and GRUs address some of these issues. Generally RNNs are difficult to train.

Some references addressing these issues:

  • Merity, Stephen, Nitish Shirish Keskar, and Richard Socher. "Regularizing and optimizing LSTM language models." arXiv preprint arXiv:1708.02182 (2017).
  • Velikovi, Petar, et al. "Graph attention networks." arXiv preprint arXiv:1710.10903 (2017).

Elman RNN

The PyTorch RNN class implements the Elman RNN directly. Below is an indirect implementation of the Elman RNN using the RNN layer. The class My_RNN, utilizes RNN to create the input-to-hidden and hidden-to-hidden weight matrices. Each call to RNN() accepts a matrix of input vectors and a matrix of hidden vectors. It returns the matrix of hidden vectors that results from one step or batched. Batched learning is achieved by training in parallel, and it is fully scalable as long as the 3D Tensor fits in the GPU memory.

In addition to controlling the input and hidden size hyperparameters in the RNN, there is a Boolean argument for specifying whether the sequence dimension T will be on the 0th dimension. This flag is present in all PyTorch RNNs implementations, as well. When set to True, the RNN swaps the 0th and 1th dimensions on the input tensor, setting the batch dimension B as first dimension and T as the second. Features dimension M comes last.

In the My_RNN class, the forward() method internally loops over the input tensor to compute the hidden state vector for each time step, guided by the sequence dimension T. Notice that there is an option for specifying the initial hidden state (i.e. layer), but if it is not provided, a default hidden state vector of all 0s is used. As the RNN class loops over the length of the input vector, it computes a new hidden state.

These hidden states are aggregated and ultimately stacked. Before they are returned, the batch_first flag is checked again. If it is True, the output hidden vectors are permuted so that the batch is once again on the 0th dimension. The output of the class is a three-dimensional tensor, i.e. there is a hidden state vector for each data point on the batch dimension and each time step. We can use these hidden vectors in many ways, depending on the task at hand. One way that we can use them is to classify each time step to some discrete set of options. This method means that the RNN weights will be adjusted to track information relevant to predictions at each time step.

Long Short-Term Memory RNN

In practice, RNNs are particularly susceptible to the vanishing gradient problem, or potentially exploding gradient due to the nature of the feedback network and a high learning rate. A new RNN type is invented as the Long Short-Term Memory (LSTM) variant of the RNN which has three gates (layers) for more control:

  • input gate
    $i_t = \sigma \left(W_{ii}x_t + b_{ii} + W_{hi}h_{t-1}+b_{hi} \right)$
    $g_t = \tanh \left(W_{ig}x_t + b_{ig} + W_{hg}h_{t-1}+b_{hg} \right)$
  • output gate
    $o_t = \sigma \left(W_{io}x_t + b_{io} + W_{ho}h_{t-1}+b_{ho} \right)$
  • forget gate
    $f_t = \sigma \left(W_{if}x_t + b_{if} + W_{hf}h_{t-1}+b_{hf} \right)$

Cell state $c_t = f_t \odot c_{t-1} + i_t \odot g_t$

Hidden state $h_t = o_t \odot \tanh c_t$

A standard RNN remembers everything forever. The LSTM forget gate allows us to model the sequences while forgetting parts of the learned information while adding some new information. The training of the LSTM cell involves learning parameters (weights) of these layers. Note that the blue and black tensors are NN layers appended in the figure on the right. Compare the block diagram with the PyTorch LSTM equations.

Predicting Language Origin of Surnames

Apply the RNN to the problem of surname classification with respect to their country of origin, i.e. character-level RNN application to text. This problem can have applications such as product recommendations while ensuring fair outcomes for users across different demographics/nationalities.

Important: Demographic and other self identifying attributes are collectively called protected attributes. We must exercise care in the use of such attributes in modeling and in products.

The following example uses the nn.RNN module to predict surname origin out of 18 classes. The problem is a difficult one as the labels are highly unbalanced and problem has numerous number of classes.

  • Problem: Predicting the origin of surnames. Given a surname which is the most likely country of origin? Note that this kind of profiling is conducted online by a variety of services such as retail, marketing, advertising, social networks, security, etc.
  • Data: Surnames dataset of 18 countries, ranging from 73 entries (Vietnamese), 139 entries (Polish), to 2000 entries (Arabic), 3668 entries (English), and 9408 entries (Russian). A percentage of the surnames exist in different countries (e.g. English vs Scottish)
  • Methods: Apply a Recurrent Neural Network model to the problem of surname classification with respect to their country of origin. Set the RNN framework to input character-level sequences.
  • The Hardest Challenge: Numerous classes, i.e. 18; Highly unbalanced dataset; RNN are very slow to train
  • Key Steps: Encode the sequences to fixed length. Penalize large classes, weigh to compensate high unbalance. Set the RNN framework. Build an Elman RNN class and create subclasses (e.g. LSTM, GRU). Use batch training for RNN

# Letter index 0 is the padding value, i.e. padding to fill up the vector to SEQ_SIZE, necessary for batched

LetterVocabulary, LetterVocabularyIndex, Index2Voc, Sequences = {' ':0}, 1, {0:' '}, {}
for fn in sorted([_ for _ in listdir(PATH_DATA) if _.endswith('.txt')]):
    lang, seqs = path.splitext(path.basename(fn))[0], []

    if lang not in LANGS:  # test case
        continue

    with open(path.join(PATH_DATA, fn), 'r', encoding="utf8") as fin:
        for row in fin.read().splitlines():
            seq = np.zeros(SEQ_SIZE, dtype=np.int32)
            for i_, letter in enumerate(row.lower()):  # Convert the surname to lower case

            seqs += [seq]
    Sequences[lang] = seqs

Elman RNN Classifier

Create the one hot encoded character sequence as a signal for every surname. Make the list of tuples of (sxx, y) in order to help to shuffle the training data which can be very important to have unbiased fast RNN training.

  • Use batch_first=True to mark the first dimension as the batch dimension of the Tensor. RNN will use the Tensor data structure dimensions $\text{batch}\times \text{sequence} \times \text{features}$ or B x T x M
  • Use 'relu' as the activation function on Elman RNN
  • Use dropout of 0.2 when 2 or more hidden layers are provided

As can be seen below, RNN learns the sequences successfully and its performance is comparable to SVM when the first letter is capitalized (left as an Exercise). Below SVM performance is achieved when the surnames are all lower case and this is a more difficult problem (Why?).

Note that the Elman RNN performance below heavily depends on the number of epochs and the batch size.

    Xy = []
    for lang in Sequences.keys():
        for seq in Sequences[lang]:
            T = SEQ_SIZE  # necessary for batched
            sxx = np.zeros((T, M))
            for i in range(T):  # for the duration of the signal
                if seq[i] > 0:

            Xy += [(torch.tensor(sxx, dtype=torch.float32),
                    torch.tensor([LANGS_CAT[lang]], dtype=torch.int64))]
    return Xy

class My_RNN(nn.Module):
    
    def __init__(self, n_hidden, n_hid_layers=1, epochs=10, eta=0.0005, batch_size=100, weight=None, info=True):
        """ A PyTorch neural network model based on RNN cell, batched """
        super(My_RNN, self).__init__()

        self.n_hidden= n_hidden  # hidden layer size
        self.n_hid_layers= n_hid_layers  # number of hidden layers
        self.epochs= epochs  # number of learning iterations
        self.eta= eta  # learning rate
        self.B= batch_size  # size of training batch - 1 would not work
        self.info= info  # debug info
        
        self.rnn, self.outlayer = None, None

        self.softmax = nn.LogSoftmax(dim=1)
        # loss function, since the last layer is nn.LogSoftmax
        self.criterion = nn.NLLLoss(weight=weight)

    def forward(self, _X, _h0):
        output, hn = self.rnn(_X, _h0)
        output = self.outlayer(output[:, -1, :])  # output is batched
        output = self.softmax(output)
        return output, hn
    
    def init_cell(self, _M):  # Create variations of our RNN by overriding init_cell
        dropout = 0.2 if self.n_hid_layers > 1 else 0
        return nn.RNN(_M, self.n_hidden, self.n_hid_layers,
                      nonlinearity='relu',
                      bias=False, batch_first=True, dropout=dropout)

    def init_hidden(self, _B):  # batch_first = True
        return torch.zeros(self.n_hid_layers, _B, self.n_hidden).to(Gpu)  # Extra dimension - batch

    def fit(self, _Xy):
        from random import shuffle
        import sys
        import torch.optim as optim

        M= _Xy[0][0].shape[1]  # number of features, based on batch input
        C= np.unique([int(_[1].data[0]) for _ in _Xy]).shape[0]  # number of class labels

        self.rnn = self.init_cell(M).to(Gpu)
        self.outlayer = nn.Linear(self.n_hidden, C).to(Gpu)
        
        self.optimizer = optim.Adam(self.parameters(), lr=self.eta)
        
        for e in range(self.epochs):
            # Shuffle the input to randomly interleave classes, note that they are tuples, i.e. (x, y)
            shuffle(_Xy)

            N = len(_Xy)
            L, totloss = 0, 0

            while L < N-self.B:
                sxx = torch.stack([_[0] for _ in _Xy[L:L+self.B]]).to(Gpu)  # get the X
                y = torch.tensor([_[1] for _ in _Xy[L:L+self.B]], dtype=torch.int64).to(Gpu)  # get the y
                output, loss = self.train_signal(sxx, y, self.B)
                
                totloss += loss
                L += self.B
                
                if self.info:
                    sys.stderr.write(f"\r{e+1:03d}/{self.epochs:4d} | Loss: {loss:6.2f} | "
                                     f"Avg loss: {totloss/(e+1):6.2f} | {y.data.tolist()[0]}")
                    sys.stderr.flush()
    
    def train_signal(self, _sxx, _y, _B):
        h0 = self.init_hidden(_B)
        self.optimizer.zero_grad()

        output, hn = self.forward(_sxx, h0)

        loss = self.criterion(output, _y)
        loss.backward()
        self.optimizer.step()
        return output, loss.item()

    def predict(self, _sxx):  # Tensor dimensions: B x T x M
        _sxx = torch.stack(_sxx)
        with torch.no_grad():
            h0 = self.init_hidden(_sxx.shape[0])  # reset the hidden layer
            output, hn = self.forward(_sxx.to(Gpu), h0)

        p_values, indices = output.max(dim=1)
        return indices.to('cpu')

LSTM Classifier

Use the My_RNN class and modify hidden layer generation and creation of the RNN layer via overriding init_hidden and init_cell methods.

Internally, LSTM last layer has default $\tanh$ activation function. It uses two hidden buffers, one for hidden state and one for cell state, and these are passed as a tuple in init_hidden.

As can be seen below LSTM classifier is performing superbly among all the classifiers tried in this notebook.

In[20]:
class My_LSTM(My_RNN):

    def __init__(self, n_hidden, n_hid_layers=1, epochs=10, eta=0.0005, batch_size=100, weight=None, info=True):
        """ A PyTorch neural network model based on LSTM RNN cell, batched """
        super(My_LSTM, self).__init__(n_hidden, n_hid_layers=n_hid_layers,
                                      epochs=epochs, eta=eta, batch_size=batch_size, weight=weight, info=info)

    def init_hidden(self, _B):  # batch_first = True
        return (torch.zeros(self.n_hid_layers, _B, self.n_hidden).to(Gpu),
                torch.zeros(self.n_hid_layers, _B, self.n_hidden).to(Gpu))

    def init_cell(self, _M):  # override
        dropout = 0.2 if self.n_hid_layers > 1 else 0
        return nn.LSTM(_M, self.n_hidden, self.n_hid_layers,
                       #nonlinearity='relu',
                       bias=False, batch_first=True, dropout=dropout)
