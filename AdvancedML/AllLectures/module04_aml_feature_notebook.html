Feature Selection and Extraction

Introduction

"If you have extracted the wrong features, no classifier will work; if you have extracted the right features, any classifier will work.", King-Sun Fu

Features are independent variables that measure a property or characteristic of a particular problem. Selecting informative, discriminative, and independent features is the most important process of any learning model.

Feature Selection

The process of selecting a subset of strong features as independent variables of the learning model is called feature selection.

  • Simplification of models to make them explainable
  • Requires less computation in training
  • Avoid the curse of dimensionality, especially an issue for probabilistic models
  • Improve data compatibility with a learning model class, such as ONNX models
  • Encode inherent symmetries present in the input space, such as in reinforcement learning

Generally a feature selection is possible because the features we collected are dependent to each other.

Feature Extraction

Feature extraction is a process of dimensionality reduction by which an initial set of raw data is reduced to more manageable groups for processing. A characteristic of these large data sets is a large number of variables such as in text, audio, or image processing. Feature extraction is the process to select and combine variables into feature vectors, effectively reducing the amount of data, while still accurately, completely describing the original dataset.

Latent Space

A latent feature space or embedding space is an embedding of a set of data points within a manifold in which items resembling each other are positioned closer to one another. The coordinates, or positions within the latent space are a new representation of those objects, effectively extracted (new) features.

Manifold learning is the process of finding a manifold which can generalize linear frameworks like PCA to be sensitive to non-linear structures in data.

Geometric surfaces can be described by manifolds. On a manifold, the data points are reachable by a linear course even though the manifold can be of higher dimensions. Such as an ant walking on a cube in a straight direction. From the point of view of the ant, everywhere it walks should look like a flat plane.

This process not only reduces the original number of dimensions but also places similar data points in higher dimensions to be close to each other in lower dimensions. Something like data compression but also clustering together. This kind of feature extraction helps a classifier.

Text Bag-of-Words

Text Analysis is a major application field for machine learning algorithms. However, the raw data, a sequence of symbols cannot be fed directly to the algorithms themselves as most of them expect numerical feature vectors with a fixed size rather than the raw text documents with variable length.

In Natural Language Processing (NLP) field, text analysis is helped by methods specific to linguistics or language. In NLP, the feature is the token delimited by white space determined by the language. Required steps for feature extraction as in the following,

  • tokenizing strings, and assigning an integer id for each possible token
  • counting the occurrences of tokens in each document
    • note that a document can be a phrase, sentence, paragraph, section, chapter, or even a book, etc.
  • normalizing and weighting with diminishing importance tokens that occur in the majority of samples/documents

Let's see an example based on TF-IDF feature extraction (i.e. Term Frequency - Inverse Document Frequency).

The measure of a feature (token/term) in a document is simply proportional to the term's frequency. However, this metric has to be normalized because the term frequency alone will tend to incorrectly emphasize documents which happen to use common and frequently used words (such as 'the', if stop-word filtered then 'news' in a news corpus), without giving enough weight to the more meaningful terms (such as 'alien' or 'inflation'). This normalization can be based on the number of the documents by counting how many documents have that term.

$\texttt{tfidf}(t,d,D) = \texttt{tf}(t,d) \cdot \texttt{idf}(t,D)$, where

$\texttt{tf}(t,d) = f_{t,d}$, which is the raw count of term $t$ in document $d$

$\texttt{idf}(t,D) = \log \frac {N}{ |\{ d \in D : t \in d \}| }$

$N=|D|$, total number of documents $d$ in the corpus of $D$

sklearn library has the method TfidfVectorizer to generate the features for a given document.

Audio Feature Extraction

A variety of acoustic features have also been explored. Such as acoustic profiles and the acoustic features of fundamental frequency/pitch (F0), energy, speech rate, and spectral information in voiced and unvoiced portions are utilized. Speech synthesis for archetypal emotions using acoustic features of (1) pitch-related features; (2) formant frequencies; (3) timing features; (4) voice-quality parameters; and (5) articulation parameters. Image Feature Extraction

A Convolutional Neural network CNN is a multiple-layer neural network that consists of convolutional layers, pooling layers, and fully connected layers as hidden layers. In a convolution layer, a filter is slid across the data, and feature maps are extracted through element-wise multiplication. In general, a convolution layer has multiple filters, where one filter creates one feature map. The role of this layer is feature extraction.

The convolution is performed by multiplying the filter pixel wise with the portion of the image, and summing the result. The output is more positive when the portion of the image closely matches the filter and more negative when the portion of the image is the inverse of the filter. If we move the filter across the entire image, from left to right and top to bottom, recording the convolutional output as we go, we obtain a new array that picks out a particular feature of the input, depending on the values in the filter.

There are several algorithms to detect features such as shaped, edges, or motion in a digital image or video.

Autoencoders

The purpose of autoencoders is unsupervised learning of efficient data encoding. Feature extraction can identify key features in the data for coding by learning from the coding of the original data set to derive new ones. In a way autoencoders can discover Lego pieces of the images, or subimages, to be used to construct larger images by linear combinations.

Autoencoders are a type of neural network which generates an "n-layer" coding of the given input and attempts to reconstruct the input using the code generated. This NN architecture is composed of the encoder structure, the decoder structure, and the latent space, also known as the "bottleneck". To learn the data representations of the input, the network is trained using given dataset, no labels needed. These compressed, data representations go through a decoding process wherein which the input is reconstructed. An autoencoder is a regression task that models an identity function.

Variational Autoencoders

If an embedding (or a latent vector) falls in the middle of two clusters, then that particular data point might be represented by an incorrect feature vector. Variational autoencoders (VAE) help resolve this problem by generating feature vectors that have a 0-mean and 1-standard-deviation.

Sparse Coding

Given a dataset, sparse coding involves learning a more efficient basis from over-complete bases. The aim of sparse coding is to find a set of basis vectors $\phi_i$ such that an input vector $x$ can be represented as a linear combination of these basis vectors, $x=\sum\limits_{i=1}^k a_i \phi_i$, such that only a few non-zero components $a_i$ are allowed, hence the sparsity.

The sparse coding cost function,

$\texttt{minimize}_{a_i^{(j)},\phi_i}\sum\limits_{j=1}^m \| x^{(j)} \sum\limits_{i=1}^k a_i^{(j)} \phi_i \|^2 + \lambda \sum\limits_{i=1}^k S(a_i^{(j)})$, where $x \in \mathbb{R}^n$

The $S(.)$ is a sparsity cost function which penalizes $a_i$ for being non-zero. The first term of the sparse coding objective as a reconstruction term which tries to force the algorithm to provide a good representation of $x$ and the second term as a sparsity penalty which forces the representation of $x$ to be sparse. The constant $\lambda$ is a scaling constant to determine the relative importance of these two components.

Techniques such as Principal Component Analysis (PCA) help learn a complete set of basis vectors efficiently, albeit lossy, without any constraint on $a_i$.

from nltk.corpus import reuters
from string import punctuation
from nltk.corpus import stopwords
from nltk import word_tokenize
import re  # we will remove some contractions with regular expression
from sklearn.feature_extraction.text import TfidfVectorizer

# Combination of stop words and punctuations
Stop_words = stopwords.words('english') + list(punctuation)

# Drop digits for this example
def tokenize(text):
    terms = word_tokenize(text)
    # all lower case
    terms = [w.lower() for w in terms]
    # filter stop words
    terms = [w for w in terms if w not in Stop_words and not w.isdigit()]
    # remove contractions, the best way might be having a list 
    terms = [w for w in terms if not re.search(r'^\W\w+$', w)]
    return terms

Let's implement a vanilla autoencoder working on the MNIST dataset. Z_DIM is the latent space. The decoded images need to be detached (detach()), to be stored in the list for visualization. The latent space dimension can be as low as 1 dimension, i.e. single perceptron. Note that every image is stored in the debug list so use the number of epochs to reach the last trained model and reconstructed image.

X_DIM= 28  # size of the images

class Autoencoder(torch.nn.Module):
    def __init__(self, z_dim):
        super().__init__()
        self.z_dim= z_dim

        self.encoder = torch.nn.Sequential(
            torch.nn.Linear(X_DIM*X_DIM, 128),
            torch.nn.ReLU(inplace=True),
            torch.nn.Linear(128, 64),
            torch.nn.ReLU(inplace=True),
            torch.nn.Linear(64, self.z_dim)
        )

        self.decoder = torch.nn.Sequential(
            torch.nn.Linear(self.z_dim, 64),
            torch.nn.ReLU(inplace=True),
            torch.nn.Linear(64, 128),
            torch.nn.ReLU(inplace=True),
            torch.nn.Linear(128, X_DIM*X_DIM),
            torch.nn.Sigmoid()  # why not Tanh?
        )

    def forward(self, _X):
        encoded = self.encoder(_X)
        decoded = self.decoder(encoded)
        return decoded

%%time

EPOCHS = 10
debug, loss_train = [], []
for e in range(EPOCHS):
    loss = None
    for (img, _) in loader:
        img = img.reshape(-1, X_DIM*X_DIM).to(gpu)
        reconstructed = ae_model(img)
        loss = loss_func (reconstructed, img)

        optim.zero_grad()
        loss.backward()
        optim.step()

        loss_train += [loss.item()]
        debug.append((e, img.cpu(), reconstructed.detach().cpu()))
    sys.stderr.write(f"\r{e+1:02d}/{EPOCHS:02d} | Loss: {loss:<6.4f}")
    sys.stderr.flush()
