Deep Reinforcement Learning¶

Introduction¶

Reinforcement learning (RL) is the concept of learning by interaction with an environment to maximize a reward function. Maximizing a reward function is also related to the concept of minimizing the loss function in supervised learning. The labels for learning a series of actions are not known or defined upfront in reinforcement learning, instead, they need to be learned through interactions with the environment to achieve a certain desired outcome, such as winning at a game. The RL model (also called an agent) interacts with its environment, and by doing so generates a sequence of interactions that are together called an episode. Through these interactions, the agent collects a series of rewards determined by the environment. These rewards can be positive or negative, and sometimes they are not disclosed to the agent until the end of an episode (e.g., the game).

Consider the game of chess with a large number of possible inputs (the states of the system), it is impossible to label each configuration or state as positive or negative. Defining a learning process involves providing rewards (or penalties) at the end of each game. The essence of RL is that we cannot or do not teach an agent how to act but only specify what we want the agent to achieve. Then a reward depending on the agent's success or failure can be set to push the agent do act better with every episode.

The reward signal is the feedback that the agent receives from interacting with the environment in the form of a positive or negative real value. The frequency at which the agent receives the reward depends on the given task or problem. A game of chess can have the reward after a full game, or solving a maze can have the reward after each time step, such as the agent tries to maximize its accumulated rewards over its lifetime (an episode).

Consider a robot as the agent. Then the state of the agent is the set of all of its variables such as current position (longitude, latitude, and altitude), velocity (as a vector), remaining battery life, the speed, etc. At each time step $t$, the agent interacts with the environment through a set of available actions $A_t$. Based on the action taken by the agent denoted by $A_t$, while it is at state $S_t$, the agent will receive a reward signal $R_{t+1}$, and its state will become $S_{t+1}$.

During the learning process, the agent must try different actions (exploration) so that it can progressively learn which actions to prefer and perform more often (exploitation) in order to maximize the total, cumulative reward.

Markov Decision Processes¶

Markov decision processes (MDP) formulate RL problems, and then they can be solved by dynamic programming. Dynamic programming may not be a feasible approach when the size of states is relatively large. Denote the agent's starting state as $S_0$, then $\{S_0,A_0,R_1\}, \{S_1,A_1,R_2\}, \{S_2, A_2, R_3\}, \dots$ is the model assuming $S_t$, $R_{t+1}$, and $A_t$ are time-dependent random variables that take values from predefined finite sets denoted by $s \in \hat{S}$, $r \in \hat{R}$, and $a \in \hat{A}$, respectively. In an MDP, these time-dependent random variables, $S_t$ and $R_{t+1}$, have probability distributions that only depend on their values at the preceding time step, $t–1$. Then,

$p(s',r \mid s,a) \overset{\text{def}}{=} P(S_{t+1}=s', R_{t+1}=r \mid S_t=s, A_t=a)$

This probability distribution completely defines the dynamics of the environment (or model of the environment).

  • Model-based, RL requires a model of the environment or try to learn a model of the environment
    • Dynamic Programming (DP)
  • Model-free
    • Monte Carlo (MC), Temporal Difference (TD), SARSA, Q-learning

The environment dynamics can be considered deterministic if particular actions for given states are always or never taken, $p(s',r \mid s,a) \in \{0,1\}$. Otherwise, the environment would have stochastic behavior (more common).

Assume a future state depends on the current state and performed action, then using all possible rewards,

$p(s' \mid s,a) \overset{\text{def}}{=}\sum\limits_{r \in R}p(s',r \mid s,a)$

This probability is called state-transition probability.

RL Terminology Return, Policy, and Value Function¶

The Return¶

The return at time $t$ is the cumulated reward obtained from the entire duration of an episode. Reminder $R_{t+1}=r$ is the immediate reward obtained after performing an action $A_t$. Then,

$G_t \overset{\text{def}}{=} R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} \dots = \sum\limits_{k=0} \gamma^k R_{t+k+1}$

$\gamma$ is the discount factor $[0,1]$ and it indicates how much the future rewards are applicable at $t$. Intuitively, the discount factor can affect the reward in the future, such as inflation or from state to state, carrying a reward to earlier states where a particular criteria is met such as the game has not lost yet.

Policy¶

Denoted by $\pi(a \mid s)$ policy is a function that determines the next action to take, which can be either deterministic or stochastic.

The learning process may have the policy updated as the agent gains more experience. The optimal policy $\pi_*(a \mid s)$ is the policy that yields the highest return.

Value Function¶

The state-value function measures the value of each state based on the return. Then,

$v_\pi(s) \overset{\text{def}}{=} \texttt{E}_\pi[G_t \mid S_t=s] = \texttt{E}_\pi\left[\sum\limits_{k=0}\gamma^{k+1}R_{t+k+1} \mid S_t=s \right]$

can model the value function of state $s$ as the expected return after following the policy $\pi$. This value function is implemented by a lookup table.

Dynamic Programming¶

Dynamic programming is about recursive problem-solving by breaking them down into smaller subproblems via memo-ization.

The Bellman equation can simplify the computation of the value function, such that rather than summing over multiple time steps,

$v_\pi(s) \overset{\text{def}}{=} \texttt{E}_\pi[G_t \mid S_t=s] = r + \gamma \texttt{E}_\pi[G_{t+1} \mid S_t=s]$

The action-value function,

$q_\pi(s,a) \overset{\text{def}}{=} \texttt{E}_\pi[G_t \mid S_t=s,A_t=a] = r + \gamma \texttt{E}_\pi[G_{t+1} \mid S_t=s,A_t=a]$

Then,

$v_\pi(s) = \sum\limits_{a\in \hat{A}} \pi(a \mid s) \sum\limits_{s' \in \hat{S}, r \in \hat{R}} p(s',r \mid s,a)[r+\gamma v_\pi(s')]$

which is called the Bellman equation simplifying the computation of the value function by eliminating the iterative loop along the time axis.

Reinforcement Learning Algorithms¶

In most problems, the environment dynamics $p(s',r \mid s,a)$ is not known. Monte Carlo, temporal difference learning, Q-learning, and deep Q-learning approaches are the solutions.

Dynamic Programming¶

Full knowledge of the environment dynamics available in the form of all transition probabilities $p(s',r \mid s,a)$.

The agent's state has the Markov property, the next action and reward depend only on the current state and the choice of action at $t$ (i.e. similar to bi-grams in NLP).

Policy Evaluation¶

Based on the Bellman equation the value function for an arbitrary policy $\pi$ can be computed. Given an initial condition as $v^{(0)}(s)$, iterate on $i$,

$v^{(i+1)}(s) = \sum\limits_a \pi(a|s) \sum\limits_{s' \in \hat{s},r \in \hat {R}} p(s',r \mid s,a)[r+\gamma v^{(i)}(s')]$

As iterations go to infinity, $v^{(i)}(s)$ converges to the true state-value function, $v_{\pi}(s)$. No need to interact with the environment as the full environment dynamics are known.

Improving the Policy using the Estimated Value Function¶

As the value function $v_{\pi}(s)$ is computed find a new policy $\pi'$ that would yield higher or at least equal value than using the current policy $\pi$, such as $v_{\pi'}(s) \ge v_{\pi}(s)$, $\forall s \in \hat{S}$.

In order to find $\pi'$ that always has a better or equal value for each state, we first compute the action-value function, $q_\pi(s,a)$, for each state $s$ and action $a$ based on the computed state value using the value function $v_\pi(s)$.

After the highest state value by evaluating all state-action pairs via $q_\pi(s,a)$ is available, compare the corresponding action with the action selected by the current policy. If the action suggested by the current policy $\underset{a}{\arg \max } \, \pi(s,a)$ is different from the action suggested by the action-value function $\underset{a}{\arg \max } \, q_\pi(s,a)$ then update the policy by reassigning the probabilities of actions to match the action that gives the highest action value $q_\pi(s,a)$. This is called the policy improvement algorithm.

Policy Iteration¶

The general policy improvement (GPI) algorithm will strictly yield a better policy, unless the current policy is already optimal which also follows notions of dynamic programming. In other words, if the problem is convex then the greedy choice of suboptimal solutions will yield a global optimum (why?).

Value Iteration¶

Combine the two tasks of policy evaluation and policy improvement into a single step as in the following.

$v^{(i+1)}(s) = \underset{a}{\max} \sum\limits_{s',r} p(s',r \mid s,a)[r+\gamma v^{(i)}(s')]$

Monte Carlo¶

Dynamic programming relies on a simplistic (and never happens) assumption that the environment's dynamics are fully known, i.e. $p(s',r \mid s,a)$.

Commonly, the state-transition probabilities of the environment are not known but can be learned by an agent to learn through interacting with the environment. Using MC methods, the learning process is based on the simulated experience.

State-value Function Estimation using MC¶

Assume that a lookup table is used for obtaining the value corresponding to the value function, $V(S_T=s)$. MC updates for estimating the value function are based on the total return obtained in that episode starting from the first time that state $s$ is visited, called first-visit Monte Carlo value prediction.

Action-value Function Estimation using MC¶

Extend the algorithm for estimating the first-visit MC state-value prediction by computing the estimated return for each state-action pair using the action-value function. Consider visits to each state-action pair $(s,a)$.

An exploratory start, which assumes that every state-action pair has a non-zero probability at the beginning of the episode is necessary. Another approach for dealing with this lack-of-exploration issue is called the $\epsilon$-greedy policy.

Finding an Optimal Policy using MC Control¶

Starting from a random policy $\pi_0$, the process of alternating between policy evaluation and policy improvement can be illustrated as,

$\pi_0 \overset{Eval}{\longrightarrow} q_{\pi_0} \overset{Improve}{\longrightarrow} \pi_1 \overset{Eval}{\longrightarrow} q_{\pi_1} \overset{Improve}{\longrightarrow} \pi_2 \dots \overset{Eval}{\longrightarrow} q_* \overset{Improve}{\longrightarrow} \pi_*$

Policy Improvement¶

Given an action-value function $q(s,a)$ generate a greedy (deterministic) policy,

$\pi(s) \overset{\text{def}}{=} \underset{a}{\arg \max} \, q(s,a)$

To avoid the lack-of-exploration problem let the non-optimal actions have a small chance $\epsilon$ to be chosen. This is called the $\epsilon$-greedy policy, all non-optimal actions at state $s$ have a minimal $\frac{\epsilon}{|A(s)|}$ probability of being selected (instead of $0$), and the optimal action has a probability of $1−\frac{(|A(s)|-1)}{|A(s)|} \epsilon$ (instead of $1$).

Temporal Difference Learning¶

TD is an improvement or extension of the MC-based RL approach by leveraging some of the learned properties to update the estimated values before reaching the end of the episode called bootstrapping. A test or metric that uses random sampling with replacement helps measure of accuracy (bias, variance, confidence intervals, prediction error, etc.) to sample estimates.

  1. Estimate the value function (value prediction)
  2. Improve the policy (control task)

TD prediction¶

At the end of each MC episode estimate the return $G_t$ and then,

$V(S_t) \leftarrow V(S_t) + \alpha(G_t-V(S_t))$

where $G_t$ is the target return to update the estimated values and $(G_t–V(S_t))$ is a correction term added to the current estimate of the value $V(S_t)$. $\alpha$ the learning rate hyperparameter.

MC uses $G_t$ which is not known until the end of the episode. Thus use $G_t$ in place of $G_{t:T}$ the return obtained at time step $t$ while considering all the events that occurred from time step $t$ until the final time step $T$.

In TD learning the actual return $G_{t:T}$ is replaced with $G_{t:t+1}$. Then,

$V(S_t) \leftarrow V(S_t) + \alpha(G_{t:t+1}-V(S_t))$

In MC $G_{t:T}$ is not available until the end of the episode, so we should execute as many steps as needed to get there. On the contrary TD needs to go one step ahead to get the target return. This is also known as TD(0).

The TD(0) algorithm can be generalized to an n-step TD algorithm, which incorporates more future steps, the weighted sum of n future steps. $n=1$ is identical to TD(0). If $n \rightarrow \infty$ then the n-step TD algorithm will be the same as the MC algorithm. The update rule for n-step TD,

$V(S_t) \leftarrow V(S_t) + \alpha(G_{t:t+n}-V(S_t))$

And,

$G_{t:t+n} \overset{\text{def}}{=} \begin{cases} R_{t+1}+\gamma R_{t+2}+\dots+\gamma^{n-1} R_{t+n}+\gamma^n V(S_{t+n}) & \text{if } t+n < T \\ G_{t:T} & \text{otherwise} \end{cases}$

Control task can be one of two algorithms, on-policy control and off-policy control. In on-policy TD control, the value function is updated based on the actions from the same policy that the agent is following, while in an off-policy algorithm, the value function is updated based on actions outside the current policy.

On-policy TD Control (SARSA)¶

Use a lookup table $Q(S_t, A_t)$ which represents the action-value function for each state-action pair. Then the SARSA algorithm,

$Q(S_t,A_t) \leftarrow Q(S_t,A_t)+\alpha[R_{t+1}+\gamma Q(S_{t+1},A_{t+1})-Q(S_t,A_t)]$

Starting from the random policy, repeatedly estimate the action-value function for the current policy and then optimize the policy using the $\epsilon$-greedy policy based on the current action-value function.

Off-policy TD control (Q-learning)¶

Combine updating the action-value function and the policy improvement steps,

$Q(S_t,A_t) \leftarrow Q(S_t,A_t)+\alpha\left[R_{t+1}+\gamma \underset{a}{\max} Q(S_{t+1},a)-Q(S_t,A_t)\right]$

Example Nim Game¶

Nim is a game of strategy between two players in which players remove items from three piles on each turn. Every turn the player must remove at least one item from exactly one of the piles. There are two different versions of the game goal:
(A.) the player who clears the last pile wins
(B.) the player who clears the last pile (by removing 1 $\le$ items) loses

For the following Nim game learning example, for simplicity, in this notebook we will consider the goal of the game to be to clear the last pile (game (A.) above).

The following is a simple implementation of the game and the learning strategy. st[i] is the state for a pile i=0,1,2. The number of piles is hard-coded to 3 and the number of max items per pile is ITEMS_MX = 10 in the following example. Note that pile symmetry is not handled in the regular RL approach.

Note that updates to the qtable should be through the strategy, but not manually as this may break the weights and weight updates the qtable that is being accumulated. A consistent weighing across the entire qtable is important.

The model is the Q-table with a format of a matrix of type float and has dimensions (#states, #actions) which covers all possible states and actions there can be for the Nim game.

The states are simply a list of the number of items in the 1st pile, the number of items in the 2nd pile, and the number of items in the 3rd pile.
e.g., st = [10,10,10] can be a starting state

The move or action is the tuple of the number of items to remove, and the pile ID.
e.g., a = (2,0) can be an action

For ease of programming and assuring the Q-table does not contain illegal moves (e.g., move (0,0) - not removing an item from a pile) the following table indexing is used:
index = pile*ITEMS_MX + move-1 $\longleftrightarrow$ pile, move = index//ITEMS_MX, index%ITEMS_MX+1

Note that indexing is 0-based and % has higher precedence over addition.

The first index part is the pile, i.e. 0, 1, 2. The second index part uses a numbering system, simply based on the number of max items per pile, i.e. using max items of 10. Examples,
index 2 $\longrightarrow$ move 3, pile 0
index 11 $\longrightarrow$ move 2, pile 1
move 10, pile 2 $\longrightarrow$ index 29

Deep RL Nim Game¶

A neural network captures the state transition matrix and all of its optimization as in the following example:

Dqmodel = nn.Sequential(
    nn.Linear(nim.PILES_N, 64),
    nn.ReLU(),
    nn.Linear(64, 32),
    nn.ReLU(),
    nn.Linear(32, nim.PILES_N*nim.ITEMS_MX))

The input size is 3, the number of piles, and the output size is possible number of actions.
